APUNTES DE GIT Y GITHUB
🛠️ Configuración y Primeros Pasos

Configuración de usuario: Siempre configurar el usuario y correo para poder usar git, así se haría: 


git config --global user.name "angCod75" 


git config --global user.email "angelo.cole75@gmail.com" 


git config --global alias.tree "comando": para darle un alias a un comando largo.


Iniciar repositorio: Para iniciar git, usamos el comando git init, esto nos creara un repositorio "oculto" y ya estaremos indicando q el proyecto actual esta trabajando con un CONTROL DE VERSIONES.


Renombrar rama: git branch -m nombreQQuiera: para cambiar el nombre de la carpeta raíz "La principal, la primera de todas".

📸 Flujo de Trabajo (Lanzar la foto)

Paso 1: git status --> para ver en que estado estan mis ficheros (si estan en "rojo" es q git los ve pero no estan listos para la foto).

Paso 2: git add nomFichero / git add . --> metemos los ficheros en la "zona verde" (staging). Es como subir a los modelos al escenario, ya estan listos para q les saquemos la fotografia.

Paso 3: git commit -m "mensaje" --> disparamos la camara. Solo se sacara la foto de los ficheros que esten en "verde". Se guarda en el historial con un nombre y fecha para siempre.

Nota sobre VIM: Si haces git commit a secas, se abre VIM. Pulsamos "i" para escribir, ":wq" para guardar y salir, o ":q!" para salir sin guardar.


Extra: git diff --> para ver que cambios hemos hecho, otra forma de ver que cambios hay antes de hacer la "foto".

🧹 Archivos Ignorados (.gitignore)

Para qué sirve: Para que no me pida que add ficheros que sabemos que son basura. Aparecerán en "transparente" y ya no saldrán en rojo en el status.


Paso 1: touch .gitignore --> para crear el fichero "mágico" donde pondremos los nombres de lo que queremos ignorar.


Paso 2: echo ".DS_Store" >> .gitignore --> para meter dentro del fichero el nombre de lo que queremos ocultar (ej: .DS_Store o apuntes.txt).


Paso 3: git add .gitignore --> para subir el propio fichero de reglas al "escenario".


Paso 4: git commit -m "mensaje" --> para guardar las reglas de ignorar para siempre en el historial.

🕰️ Historial y Viajes en el Tiempo
Ver historial:


git log: para ver todos mis commits.


git log --graph --decorate --all --oneline: versión más bonita.


git reflog: es como el log pero avanzado, nos muestra TODO el historial no solo hasta donde esta el Head.

Regresar/Resetear:


git checkout nomFichero.x: para volver a la ultima "fotografía".

git checkout <ID-commit>: sirve para ver como estaba el proyecto en el pasado. Los ficheros de mi carpeta cambian de verdad.


git reset: vuelve para atras pero borrando el commit. Lvl de fuerza:

--soft: Nada del código. Solo borra el commit. Para corregir el mensaje o añadir archivos.

--mixed: Saca los archivos de la "zona verde" (staging). Los cambios siguen ahí pero vuelves a elegir qué subir.

--hard: BORRA TODO (commits y cambios). "Metí la pata al máximo, borra todo y déjalo como estaba antes". (Úsalo con cuidado) .

🌿 Ramas (Branches) y Fusión (Merge)
Gestión de ramas:

git branch: sirve para ver el listado de todas las ramas creadas. La que tiene el asterisco * es donde estoy.


git branch nombre-rama: para crear una rama nueva (dimensión paralela).


git switch nombre-rama: para saltar a esa rama.


git switch -c nombre-nueva: atajo para crear y saltar a la vez.


git switch main: para volver rápido a mi rama principal (el tronco).

Borrar ramas:

git branch -d nombre-rama: borrado seguro. Solo deja si ya hiciste merge.

git branch -D nombre-rama: borrado "a la fuerza". Útil para experimentos que salieron mal.


Fusión (Merge): Es el momento en el que decides unir una rama al tronco principal (main).


Regla de oro: Siempre tienes que estar parado en la rama que va a recibir los cambios (el destino).


Paso 1: git switch main.


Paso 2: git merge nombre-de-la-rama.

Conflictos: Sucede cuando he modificado la misma línea en dos ramas a la vez. Git no sabe cuál elegir y se para.

Abro el fichero y busco las marcas <<<< HEAD y >>>>.

Borro a mano la versión que no quiero y las marcas de git.


git add y git commit para terminar la fusión.


Botón del pánico: git merge --abort: deshace el intento de unión y deja todo como estaba.

📦 El Cajón Temporal (Stash)
git stash: sirve para guardar mis cambios en un cajón temporal sin hacer commit. Git limpia mi carpeta y me deja moverme de rama sin errores. (Solo guarda archivos que git ya conoce) .


git stash list: para ver todos los "paquetes" guardados (ej: stash@{0}).


git stash pop: saca los cambios del cajón, los pone en mis archivos y los borra del cajón.


git stash apply: igual que el pop, pero no los borra del cajón.


git stash drop: para tirar a la basura lo que hay en el cajón.


git stash clear: para vaciar el cajón por completo.

🏷️ Etiquetas (Tags)

git tag -a v1.0 -m "mensaje": para crear una etiqueta "anotada" (como un póstit en una foto).


git tag: para ver el listado de etiquetas creadas.


git show v1.0: para ver los detalles de esa etiqueta.


git checkout v1.0: para viajar directamente a esa versión sin usar el ID largo.

💻 Comandos Básicos (Terminal/MAC)

pwd: para ver la ruta donde nos encontramos actualmente.

cls (o clear): para limpiar la pantalla de la terminal.


touch nom.txt: para crear un txt vacio.


nano nom.txt: para crear un txt con texto.


cat > nom.txt: para crear y escribir rápidamente (cmd + d para guardar).


rm nom.xxx: para borrar.
